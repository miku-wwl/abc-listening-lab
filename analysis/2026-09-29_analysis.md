- 现场问答带来 speaker drift：提问、复述问题、回答、玩笑、demo 操作经常连在一起。

## 7. 复盘问题

1. **本次材料主要是什么类型？**  
   答：GitHub/Microsoft Build 后续技术 meetup，主要讨论 GitHub Copilot app/CLI、Code Quality、worktrees、skills、sandbox、Copilot SDK、Microsoft IQ/Foundry 和 agent evaluation。

2. **为什么 Code Quality 的费用段容易听错？**  
   答：因为价格 `$10 US` 只是表层信息，真正决定含义的是 `per active committer`、`repositories that have it enabled`、`last 90 days` 和是否使用 AI credits。

3. **worktree 和 branch 在本次语境中的区别是什么？**  
   答：branch 是版本线；worktree 允许同一仓库在本地多个路径同时 checkout 不同工作内容，适合多个 agent session 或多个 feature 并行。

4. **security review command 的核心是什么？**  
   答：它本质上是预定义 prompt，用来快速发起比较完整的安全审查，避免每次都手写复杂提示词。

5. **sandbox command 主要解决什么风险？**  
   答：限制 coding agent 执行 shell 命令时的文件系统和网络访问，避免 agent 越权读取、误删或访问不该访问的资源。

6. **Foundry IQ 在本次解释中承担什么角色？**  
   答：它被描述为企业知识层，能够把 SharePoint、Confluence、文档、存储等知识统一到 MCP server 后面，并加上权限治理。

7. **为什么 web search 和 GitHub issues 要当作 untrusted source？**  
   答：这些内容可能包含恶意文本，例如 `ignore all previous prompts` 这类 prompt injection，agent 不能直接信任外部输入。

8. **本次最值得训练的听力动作是什么？**  
   答：每听到一个产品名或命令名，就立刻补三项：它属于哪个工具层、解决什么问题、有什么限制或费用边界。
