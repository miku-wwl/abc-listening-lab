## 3. 新问题 vs 旧问题

### 重复出现的问题

1. 技术专名、缩写和产品/方法名容易被 ASR 或大脑吸附为普通词：`wireless sensor network` 被写成 `wirelessness/central network`，`SensorTag`、`LaunchPad`、`Raspberry Pi`、`IoT` 和讲师姓名不稳。
2. 长句中对象、动作、条件和结果混在一起，容易只记住最后一个术语。
3. 屏幕演示中的 `this / here / that one / the first one` 缺少画面后很难恢复指代。
4. 数字、节点、版本、概率或课程规则需要和所属对象绑定，不能只记数字本身。
5. 本次仍无裸听 notes，不能把“材料中的高风险点”写成“你本人听错的点”。

### 新出现的问题

- 技术概览与 assessment 规则要分栏记录；音频故障时同学侧聊不属于正式授课内容；团队技能组合本身是项目要求。
- 本次主题要求先建立概念图或流程图，再做逐句精听；只背孤立术语收益很低。

### 已经改善的问题

目前材料不足，暂时无法判断真实听力改善情况。连续缺少裸听 notes，只能确认仓库的分类和复盘结构更加稳定，不能据此推断声音识别能力已经提高。
